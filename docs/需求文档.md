## 1. 项目概述

### 1.1 平台定位

Sisyphus Case Platform（代号 Sisyphus-Y）是一套 AI 驱动的企业级功能测试用例自动生成平台，专为数据中台、数据湖仓、实时计算等复杂系统场景设计。

核心价值：通过显式拆分「**测什么**」（测试点/场景地图）与「**怎么测**」（用例步骤），在评审节点前置拦截方向错误，将测试效率提升 70% 以上。

### 1.2 核心工作流

```
子产品录入 → 迭代创建 → 需求录入
     ↓
需求分析（AI 诊断 + 苏格拉底追问）
     ↓
场景地图生成（AI 场景补全 + 人工确认）
     ↓
【测试点评审确认】← 关键评审节点
     ↓
对话式用例生成（SSE 流式输出）
     ↓
用例库管理（CRUD / 导出 / 版本管理）
```

## 2. 核心理念

### 2.1 两阶段评审机制

| 阶段 | 核心问题 | 输出 | 修改成本 |
|---|---|---|---|
| 场景地图（M04） | **测什么？** 覆盖哪些场景 | 测试点节点 | 低（改一行描述） |
| 用例生成（M05） | **怎么测？** 具体步骤和数据 | 完整测试用例 | 高（重写多条用例） |

### 2.2 数据持久化原则

所有 AI 生成结果必须持久化到 PostgreSQL，页面刷新后能完整恢复：

| AI 生成内容 | 持久化目标 |
|---|---|
| 诊断对话 AI 响应 | `DiagnosisChatMessage` → `DiagnosisRisk` |
| 测试点草稿 | `GenerationMessage` → `TestPoint` |
| 用例生成流 | SSE 完成后 → `TestCase` |

## 6. M03 需求分析台

路由：`/analysis`

### 6.1 主页布局

- 左栏：产品/迭代树形导航 + 需求列表（搜索/状态筛选）
- 右侧：需求详情 / AI 分析 / 覆盖追踪 三标签页

### 6.2 AI 诊断

1. 点击「开始诊断」→ AI 流式扫描需求
2. 生成结构化风险清单（歧义/缺失/矛盾/边界）
3. 苏格拉底式多轮追问对话
4. 诊断报告持久化（刷新可恢复）

**风险等级：** 高（逻辑矛盾）/ 中（边界模糊）/ 低（描述建议）

## 8. M05 用例生成工作台

路由：`/workbench`

### 8.1 两步骤流程

**Step 1 — 确认测试点：**
- 测试点分组列表（按组确认/取消）
- 右侧场景地图预览
- 有未处理红色节点时「进入生成」置灰

**Step 2 — 对话式生成：**
- 三栏布局：导航+上下文 / 对话流 / 用例预览
- 多轮对话追加生成
- 生成完成可「返回 Step 1」补充

### 8.2 生成模式

| 模式 | 说明 |
|---|---|
| 标准生成 | 基于已确认测试点 |
| 追加生成 | 在现有基础上追加 |
| 边界用例 | 专注边界和异常 |
| 自定义指令 | 自由 Prompt 输入 |

### 8.3 SSE 流式输出

- 思考过程（Thinking）可折叠
- StreamCursor 闪烁动画
- `done` 事件触发后端自动保存

### 8.4 用例 Card

每条用例包含：前置条件 / 测试步骤（有序列表）/ 预期结果 / 优先级&复杂度 Badge

## 17. 全局能力

### 17.1 全局搜索

- 快捷键 `Cmd/Ctrl + K` 唤起
- 搜索：需求 / 用例 / 知识库文档
- 键盘导航（↑↓ / ↵ / ESC）

### 17.2 用户认证

- 登录（用户名/密码），JWT Token
- 登出（用户头像菜单）

### 17.3 深色主题

- 系统默认深色主题（`#0d0f12` 背景）
- 完整 `sy-*` 设计 Token 体系

### 17.4 微交互

- SSE 流式 StreamCursor 闪烁
- 按钮 Loading 动画
- 骨架屏（TableSkeleton）
- Toast 通知
- 空状态（EmptyState）

### 17.5 软删除机制

所有核心业务数据通过 `SoftDeleteMixin` 软删除（`deleted_at` 字段），通过回收站可恢复。
