## 3. 环境搭建

### 3.1 前置条件

| 工具 | 用途 | 安装 |
|---|---|---|
| **Docker** + Docker Compose 2.0+ | 基础设施（PostgreSQL/Redis/Qdrant/MinIO） | [docker.com](https://docker.com) |
| **uv** | Python 环境 & 依赖管理 | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| **bun** | Node.js 运行时 & 包管理 | `curl -fsSL https://bun.sh/install \| bash` |
| AI API Key | LLM 服务 | ZhiPu（默认）或 OpenAI |

### 3.2 一键启动（推荐）

```bash
git clone https://github.com/koco-co/Sisyphus-case-platform.git
cd Sisyphus-case-platform
bash ./init.sh
```

`init.sh` 会自动完成以下步骤：

1. 检查并安装 `uv`、`bun`、`docker` 依赖
2. 清理占用端口（5432/6379/6333/9000/9001/8000/3000）
3. 从 `.env.example` 生成 `.env` 配置
4. 安装后端依赖（`uv sync`）和前端依赖（`bun install`）
5. 启动 Docker 基础设施（PostgreSQL/Redis/Qdrant/MinIO）
6. 执行数据库迁移（Alembic）和种子数据
7. 启动后端服务（uvicorn）、Celery Worker 和前端开发服务器
8. 验证各服务健康状态

### 3.3 手动分步启动

#### 3.3.1 启动基础设施

```bash
docker compose -f docker/docker-compose.yml up -d
```

Docker Compose 包含 4 个服务：

| 服务 | 端口 | 用途 |
|---|---|---|
| PostgreSQL 16 | 5432 | 业务数据库 |
| Redis 7 | 6379 | 缓存 / 任务队列 |
| Qdrant | 6333, 6334 | 向量数据库 |
| MinIO | 9000, 9001 | 对象存储 |

#### 3.3.2 后端

```bash
cd backend
uv sync --all-extras                    # 安装依赖
uv run alembic upgrade head             # 数据库迁移
uv run python scripts/seed.py           # 初始化种子数据
uv run uvicorn app.main:app --reload --port 8000   # 启动 API 服务
```

在另一个终端启动 Celery Worker：

```bash
cd backend
uv run celery -A app.worker.celery_app worker --loglevel=info
```

后端启动后可访问：
- API 文档：http://localhost:8000/docs（开发模式）
- 健康检查：http://localhost:8000/health

#### 3.3.3 前端

```bash
cd frontend
bun install                             # 安装依赖
bun dev                                 # 启动开发服务器
```

前端访问地址：http://localhost:3000

## 4. 开发规范

### 4.1 Python 规范

**命名约定**：

| 对象 | 风格 | 示例 |
|---|---|---|
| 文件名 | `snake_case` | `stream_adapter.py` |
| 变量/函数 | `snake_case` | `get_thinking_stream()` |
| 类名 | `PascalCase` | `DiagnosisReport` |
| 常量 | `UPPER_SNAKE_CASE` | `JWT_ALGORITHM` |

**代码质量工具**：

```bash
uv run ruff check .            # Lint 检查
uv run ruff format .           # 代码格式化
uv run pyright app/            # 类型检查（standard 模式）
```

**模块开发规范**：

每个业务模块必须包含 4 个核心文件：

- **`router.py`** — API 路由，只做参数校验和调用 service，不写业务逻辑
- **`models.py`** — SQLAlchemy ORM 模型，继承 `Base` + `SoftDeleteMixin`
- **`schemas.py`** — Pydantic 请求/响应 Schema，用于参数验证和序列化
- **`service.py`** — 业务逻辑层，接收 `AsyncSession` 作为参数

**示例 — router.py**：

```python
from fastapi import APIRouter, Depends
from sqlalchemy.ext.asyncio import AsyncSession
from app.core.dependencies import get_db
from . import service, schemas

router = APIRouter(prefix="/products", tags=["products"])

@router.get("/", response_model=list[schemas.ProductResponse])
async def list_products(db: AsyncSession = Depends(get_db)):
    return await service.list_products(db)
```

**示例 — service.py**：

```python
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select
from .models import Product

async def list_products(db: AsyncSession) -> list[Product]:
    result = await db.execute(
        select(Product).where(Product.deleted_at.is_(None))
    )
    return list(result.scalars().all())
```

### 4.2 TypeScript 规范

**命名约定**：

| 对象 | 风格 | 示例 |
|---|---|---|
| 变量/函数 | `camelCase` | `useSSEStream()` |
| React 组件 | `PascalCase` | `ChatBubble.tsx` |
| 类型/接口 | `PascalCase` | `ProductResponse` |
| 文件名（组件） | `PascalCase` | `ThinkingStream.tsx` |
| 文件名（工具） | `kebab-case` | `api-client.ts` |

**代码质量工具**：

```bash
bunx biome check .             # Lint + Format 检查
bunx biome check --write .     # 自动修复
bunx tsc --noEmit              # 类型检查
```

### 4.3 API 路由规范

- 路径使用 **`kebab-case`**：`/api/scene-map/generate`
- 遵循 RESTful 约定：
  - `GET` — 查询
  - `POST` — 创建 / 触发操作
  - `PATCH` — 部分更新
  - `DELETE` — 删除（软删除）
- 流式接口使用 SSE：路径以 `/stream` 结尾

### 4.4 数据库命名规范

- 表名：**`snake_case` 复数**，如 `test_cases`、`scene_nodes`
- 字段名：**`snake_case`**，如 `created_at`、`requirement_id`
- 外键：`{关联表单数}_id`，如 `product_id`、`iteration_id`

### 4.5 Git 规范

**分支命名**：

```

docs/<desc>               # 文档，如 docs/api-spec

```

**Commit 规范**（Conventional Commits）：


### 4.6 软删除规范

所有核心业务表通过 `SoftDeleteMixin` 支持软删除：

```python
# backend/app/shared/base_model.py

class SoftDeleteMixin:
    deleted_at = Column(TIMESTAMPTZ, nullable=True, default=None)
```

**关键规则**：

- 删除操作：`UPDATE SET deleted_at = now()`，不使用 `DELETE`
- **所有查询必须过滤**：`WHERE deleted_at IS NULL`
- 恢复：将 `deleted_at` 设为 `NULL`（通过回收站模块 M21）

```python
async def get_thinking_stream(messages: list[dict]) -> AsyncGenerator[str, None]:
    provider = settings.llm_provider
    if provider == "openai":

        async for chunk in anthropic_thinking_stream(messages):
```

### 5.3 SSE 流式协议

所有 AI 生成接口使用 Server-Sent Events 协议推送结果：

**事件类型**：

| 事件 | 数据格式 | 说明 |
|---|---|---|
| `event: thinking` | `data: {"text": "..."}` | AI 思考过程（可选展示） |
| `event: content` | `data: {"text": "..."}` | 生成内容（增量推送） |
| `event: done` | `data: {}` | 生成完成 |

**前端接收示例**（`hooks/use-sse.ts`）：

```typescript
export function useSSE<T>() {
  const [data, setData] = useState<T[]>([]);
  const [isStreaming, setIsStreaming] = useState(false);

  const start = useCallback((url: string) => {
    setIsStreaming(true);
    const source = new EventSource(url);
    source.onmessage = (event) => {
      const parsed = JSON.parse(event.data) as T;
      setData((prev) => [...prev, parsed]);
    };
    source.onerror = () => {
      source.close();
      setIsStreaming(false);
    };
  }, []);

  return { data, isStreaming, start, stop };
}
```

**Zustand 流式状态管理**（`stores/stream-store.ts`）：

```typescript
interface StreamStore {
  thinkingText: string;    // AI 思考过程文本
  contentText: string;     // 生成内容文本
  isStreaming: boolean;    // 是否正在流式传输
  isThinkingDone: boolean; // 思考阶段是否结束

  reset(): void;
  appendThinking(text: string): void;
  appendContent(text: string): void;
  setDone(): void;
}
```
