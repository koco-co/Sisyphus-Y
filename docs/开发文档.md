## 3 环境搭建

### 3.1 前置要求

- **Python 3.12+** + **uv** (包管理)
- **Node.js 20+** + **Bun** (前端运行时)
- **Docker** + **Docker Compose** (基础设施)
- **Git**

### 3.2 一键启动

```bash
bash ./init.sh
```

此脚本会依次:
1. 清理占用端口 (3000, 8000)
2. 启动 Docker 基础设施 (PostgreSQL, Redis, Qdrant, MinIO)
3. 安装后端依赖 (`uv sync`)
4. 执行数据库迁移 (`alembic upgrade head`)
5. 运行种子数据 (`python scripts/seed.py`)
6. 启动 Celery Worker
7. 启动后端 (`uvicorn`, port 8000)
8. 安装前端依赖 (`bun install`)
9. 启动前端 (`bun dev`, port 3000)

### 3.3 手动启动

```bash
# 1. 基础设施
docker compose -f docker/docker-compose.yml up -d

# 2. 后端
cd backend
uv sync --all-extras
uv run alembic upgrade head
uv run python scripts/seed.py
uv run uvicorn app.main:app --reload --port 8000

# 3. 前端
cd frontend
bun install
bun dev
```

## 4 后端开发

### 4.1 模块结构

每个业务模块遵循 DDD 四文件结构:

```
modules/<name>/
├── __init__.py
├── router.py    # API 路由 — 参数校验 + 调用 service
├── models.py    # SQLAlchemy ORM 模型
├── schemas.py   # Pydantic 请求/响应 DTO
└── service.py   # 业务逻辑 — 接收 AsyncSession
```

**关键规则**:
- `router.py` 只做参数校验和调用 service，不包含业务逻辑
- `service.py` 接收 `AsyncSession` 作为参数，不自行创建数据库连接
- `models.py` 继承 `Base` 并使用 `SoftDeleteMixin`
- `schemas.py` 使用 Pydantic v2 (`model_config = ConfigDict(from_attributes=True)`)

### 4.2 新增模块示例

```python
# modules/example/models.py
from app.models.base import Base, SoftDeleteMixin

class Example(Base, SoftDeleteMixin):
    __tablename__ = "examples"
    id = Column(Integer, primary_key=True)
    name = Column(String(100), nullable=False)

# modules/example/schemas.py
from pydantic import BaseModel, ConfigDict

class ExampleResponse(BaseModel):
    model_config = ConfigDict(from_attributes=True)
    id: int
    name: str

# modules/example/service.py
from sqlalchemy.ext.asyncio import AsyncSession

async def list_examples(db: AsyncSession):
    result = await db.execute(
        select(Example).where(Example.deleted_at.is_(None))
    )
    return result.scalars().all()

# modules/example/router.py
from fastapi import APIRouter, Depends
from app.core.deps import get_db

router = APIRouter(prefix="/api/examples", tags=["examples"])

@router.get("/", response_model=list[ExampleResponse])
async def list_all(db: AsyncSession = Depends(get_db)):
    return await list_examples(db)
```

然后在 `app/main.py` 注册路由:

```python
from app.modules.example.router import router as example_router
app.include_router(example_router)
```

### 4.3 软删除

所有核心业务表通过 `SoftDeleteMixin` 支持软删除:

```python
class SoftDeleteMixin:
    deleted_at = Column(DateTime, nullable=True, default=None)

# 查询时必须过滤
select(Model).where(Model.deleted_at.is_(None))

# 删除时设置 deleted_at
model.deleted_at = datetime.utcnow()
```

### 4.4 AI 流式输出

AI 对话使用 SSE (Server-Sent Events) 流式输出:

```python
# backend/app/ai/stream_adapter.py

async def get_thinking_stream(requirement_content: str, prompt: str):
    """根据配置的 LLM provider 返回 SSE 流"""
    provider = settings.llm_provider
    if provider == "zhipu":
        return zhipu_thinking_stream(requirement_content, prompt)
    elif provider == "openai":
        return openai_thinking_stream(requirement_content, prompt)
    # ...
```

SSE 事件格式:

```
event: thinking
data: {"delta": "正在分析需求文档..."}

event: content
data: {"delta": "## 诊断结果\n\n1. 完整性评分: 85/100"}

event: done
data: {}
```

前端消费:

```typescript
const es = new EventSource('/api/diagnosis/stream');
es.addEventListener('thinking', (e) => { /* 显示思考过程 */ });
es.addEventListener('content', (e) => { /* 显示结果内容 */ });
es.addEventListener('done', () => { es.close(); });
```

### 4.5 ZhiPu AI 集成

使用 `zhipuai` SDK (v2.1.5):

```python
from zhipuai import ZhipuAI
import asyncio

client = ZhipuAI(api_key=settings.zhipu_api_key)

async def zhipu_thinking_stream(content: str, prompt: str):
    messages = [
        {"role": "system", "content": "你是测试专家..."},
        {"role": "user", "content": f"需求:\n{content}\n\n{prompt}"}
    ]
    # zhipuai SDK 是同步的，需要用 asyncio.to_thread 包装
    response = await asyncio.to_thread(
        client.chat.completions.create,
        model=settings.zhipu_model,
        messages=messages,
        stream=True
    )
    async def generate():
        yield 'event: thinking\ndata: {"delta": "正在分析..."}\n\n'
        async for chunk in async_iter(response):
            delta = chunk.choices[0].delta.content or ""
            yield f'event: content\ndata: {{"delta": "{delta}"}}\n\n'
        yield 'event: done\ndata: {}\n\n'
    return generate()
```

## 10 常见问题

### Q: init.sh 启动失败怎么办?

1. 确认 Docker Desktop 正在运行
2. 检查端口是否被占用: `lsof -i :3000 -i :8000`
3. 检查 `.env` 文件是否存在
4. 查看日志: `cat logs/*.log`

### Q: AI 流式输出不工作?

1. 确认 `.env` 中 `ZHIPU_API_KEY` 已设置
2. 确认 `LLM_PROVIDER=zhipu`
3. 测试 API: `curl http://localhost:8000/api/health`
4. 检查 ZhiPu API 额度是否充足

### Q: 数据库迁移失败?

```bash
cd backend
# 查看当前迁移状态
uv run alembic current
# 如果需要重置
uv run alembic downgrade base
uv run alembic upgrade head
```

### Q: 如何切换 AI 模型?

修改 `.env`:
```env
# 使用 OpenAI
LLM_PROVIDER=openai
OPENAI_API_KEY=sk-xxx

# 使用 ZhiPu
LLM_PROVIDER=zhipu
ZHIPU_API_KEY=xxx

# 使用本地 Ollama
LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://localhost:11434
```
