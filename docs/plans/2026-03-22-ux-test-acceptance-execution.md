# Sisyphus-Y 全量 UX 测试验收执行计划

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 通过 Claude in Chrome MCP 对 Sisyphus-Y 平台进行全量用户视角测试，记录所有 UX 问题并输出带改进理由的综合审计报告到 `docs/ux-audit-2026-03-22.md`。

**Architecture:** 4 个场景剧本（S1 初体验 → S2 核心链路 → S3 边界异常 → S4 辅助扫查），每步截图 + 记录观察，最终汇总为结构化 UX 审计报告。

**Tech Stack:** Claude in Chrome MCP（浏览器自动化）、Next.js 前端（`localhost:3000`）、FastAPI 后端（`localhost:8000`）

---

## 执行规范

- 每个 Task 开始前：先截图查看当前页面状态
- 发现问题时：立即记录到 `session_issues` 临时列表（在 SQL DB 或内存中）
- 每个场景结束后：整理本场景发现的问题
- 最终 Task：汇总所有问题，生成 `docs/ux-audit-2026-03-22.md`

**问题记录模板**（每个发现立即记录）：
```
[UX-XXX] 标题 | 位置 | 触发步骤 | 问题描述 | 用户影响 | 改进建议 | 改进理由 | 优先级(高/中/低)
```
