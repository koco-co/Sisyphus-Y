## 一、项目背景

**Sisyphus-Y** 是 AI 驱动的企业级功能测试用例自动生成平台，面向数据中台场景。

核心主链路：需求录入 → AI 分析/追问 → 场景地图确认 → 测试点勾选 → AI 生成用例 → 用例库管理

---

## 二、技术栈

| 层 | 技术 | 关键约束 |
|---|---|---|
| 前端 | Next.js 16 App Router + TypeScript + Tailwind CSS | 包管理用 **bun**（禁止 npm/yarn） |
| 组件库 | shadcn/ui + lucide-react | 禁止用 emoji 作 UI 元素 |
| 后端 | FastAPI + Python 3.12 + SQLAlchemy 2.0 async | 包管理用 **uv**（禁止 pip） |
| 数据库 | PostgreSQL 15（软删除：deleted_at 字段） | |
| 对象存储 | MinIO | bucket: sisyphus-images, sisyphus-docs |
| 向量库 | Qdrant | collection: sisyphus_cleaned_cases |
| 缓存/队列 | Redis + Celery | |

## 三、关键开发规范（必读）

### 设计 Token（前端）
**禁止**使用裸 Tailwind 色名或硬编码 HEX，必须使用 `sy-*` token：

```
sy-bg / sy-bg-1 / sy-bg-2 / sy-bg-3
sy-border / sy-border-2
sy-text / sy-text-2 / sy-text-3
sy-accent / sy-accent-2
sy-warn / sy-danger / sy-info / sy-purple
```

错误示例：`bg-red/10 text-amber bg-blue/10 border-border2`
正确示例：`bg-sy-danger/10 text-sy-warn bg-sy-info/10 border-sy-border-2`

### 代码质量命令（每次提交前必须全部通过）
```bash
# 前端
cd frontend
bunx biome check .           # lint + format 检查
bunx tsc --noEmit            # 类型检查

# 后端
cd backend
uv run ruff check .          # lint
uv run ruff format .         # 格式化
uv run pyright app/          # 类型检查（standard 模式）
```

### 不可变原则
创建新对象，不直接 mutate：
```typescript
// 错误
obj.prop = value
// 正确
return { ...obj, prop: value }
```

### 文件大小
- 单文件 ≤ 800 行，典型 200~400 行
- 函数 ≤ 50 行

## 六、验收方式

所有任务的详细验收步骤和期望结果在 `progress.json` 中（TASK-150 ~ TASK-162）。

每完成一项任务：
1. 运行 lint + typecheck 确认通过
2. 在 `progress.json` 对应 TASK 中填写 `evidence`，将 `status` 改为 `"passed"`，填写 `passed_at`
3. 如有 Bug 则填写 `bug_description` 和 `fix_commit`，`status` 改为 `"failed"`
