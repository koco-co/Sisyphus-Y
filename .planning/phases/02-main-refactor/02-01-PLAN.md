    1. 在 `DiagnosisRisk` model (models.py) 新增字段：
       `confirmed: Mapped[bool] = mapped_column(Boolean, nullable=False, server_default="false", default=False)`

    2. 在 `DiagnosisRiskResponse` schema (schemas.py) 新增字段：
       `confirmed: bool = False`

    3. 在 `DiagnosisService` (service.py) 新增方法：
       ```python
       async def confirm_risk(self, risk: DiagnosisRisk) -> DiagnosisRisk:
           risk.confirmed = True
           await self.session.flush()
           await self.session.refresh(risk)
           return risk
       ```
       同时新增 `async def get_risk(self, risk_id: uuid.UUID) -> DiagnosisRisk | None`，查询时过滤 `.where(DiagnosisRisk.deleted_at.is_(None))`。

    4. 在 router.py 新增端点（在现有 router 路由下方追加）：
       ```python
       @router.patch("/risks/{risk_id}/confirm", response_model=DiagnosisRiskResponse)
       async def confirm_risk(risk_id: uuid.UUID, session: AsyncSessionDep) -> DiagnosisRiskResponse:
           svc = DiagnosisService(session)
           risk = await svc.get_risk(risk_id)
           if not risk:
               raise HTTPException(status_code=404, detail="DiagnosisRisk not found")
           risk = await svc.confirm_risk(risk)
           return DiagnosisRiskResponse.model_validate(risk)
       ```

    5. 生成 Alembic migration（在 backend/ 目录下运行）：
       `uv run alembic revision --autogenerate -m "add_confirmed_to_diagnosis_risks"`
       验证生成的迁移文件包含 add_column confirmed，然后运行：
       `uv run alembic upgrade head`

    注意：检查 DiagnosisRisk model 中风险级别字段实际名称（model 用 `level` 还是 `severity`），确保 schema 序列化与 model 字段名一致，ANA-07 前端将使用该字段判断高风险。

    1. 在 schemas.py 新增：
       ```python
       class RagPreviewRequest(BaseModel):
           test_point_ids: list[uuid.UUID]

       class RagPreviewResult(BaseModel):
           title: str
           score: float
           content: str

       class RagPreviewResponse(BaseModel):
           results: list[RagPreviewResult]
       ```

    2. 在 service.py 新增方法 `get_test_points_by_ids`：
       查询 `TestPoint` 表，过滤 `deleted_at.is_(None)` 且 `id.in_(ids)`，返回列表。

    3. 在 router.py 新增端点：
       ```python
       @router.post("/{requirement_id}/rag-preview", response_model=RagPreviewResponse)
       async def rag_preview(
           requirement_id: uuid.UUID,
           data: RagPreviewRequest,
           session: AsyncSessionDep,
       ) -> RagPreviewResponse:
           if not data.test_point_ids:
               return RagPreviewResponse(results=[])
           svc = SceneMapService(session)
           points = await svc.get_test_points_by_ids(data.test_point_ids)
           query = " ".join(p.title for p in points)
           try:
               from app.engine.rag.retriever import retrieve_similar_cases
               results = await retrieve_similar_cases(query, top_k=5, score_threshold=0.72)
               return RagPreviewResponse(results=[
                   RagPreviewResult(
                       title=r.metadata.get("title", ""),
                       score=r.score,
                       content=r.content,
                   )
                   for r in results
               ])
           except Exception:
               import logging
               logging.getLogger(__name__).warning("RAG preview failed, returning empty results")
               return RagPreviewResponse(results=[])
       ```

    4. 创建测试文件 `tests/unit/test_scene_map/test_rag_preview.py`（如目录不存在先创建 `__init__.py`），测试 Qdrant 不可达时的降级行为，使用 `unittest.mock.patch` mock `retrieve_similar_cases` 抛出异常。
