  <action>
    审查脚本结构（参考 vectorize_testcases.py 骨架）：

    常量：
    - REVIEW_SYSTEM: 审查 Prompt（5 条规则 + JSON 输出格式 + 丢弃原因分类）
    - REPORT_FILE = scripts/review_report.json
    - PROGRESS_FILE = scripts/.review_progress.json

    函数：
    - read_csv_cases(path, sample) -> list[dict]  # utf-8-sig BOM 处理
    - make_point_id(file_path, row_index) -> str  # md5 稳定 ID
    - csv_row_to_text(row) -> str  # 用于嵌入的文本，格式：用例: {标题}\n前置条件: ...\n步骤: ...\n预期: ...
    - review_case(case_text) -> dict  # async，调用 invoke_llm，安全 JSON 提取，返回 {"verdict": "pass"|"polish"|"discard", "polished"?: {...}, "discard_reason"?: str}
    - rebuild_knowledge_chunks() -> int  # 从 KnowledgeDocument DB 读取，重新向量化
    - main(sample, dry_run) -> None  # 主流程

    main() 流程：
    1. recreate_collection(TESTCASE_COLLECTION)
    2. recreate_collection(COLLECTION_NAME)  # knowledge_chunks
    3. rebuild_knowledge_chunks()
    4. 遍历「清洗后数据」目录下所有 .csv 文件（glob "**/*.csv"）
    5. 加载 .review_progress.json（幂等 ID 集合）
    6. 对每行：make_point_id → 跳过已处理 → csv_row_to_text → review_case
    7. pass/polish → embed + upsert；discard → 记录原因
    8. 打印进度（每 50 条）
    9. 保存 review_report.json
    10. tabulate 打印汇总表格

    CLI 参数：
    - --sample N：每个 CSV 只取前 N 行
    - --dry-run：不实际调用 LLM（用于测试连通性），只打印读取到的文件数

    rate limit 处理：每条审查后 asyncio.sleep(0.3)，捕获 429 后 sleep(10) 再重试一次。
    JSON 提取：re.search(r'\{.*\}', content, re.DOTALL)，失败时 verdict = "discard" + "无法修复"。

    rebuild_knowledge_chunks() 实现：
    - 需要 DB session（用 asyncpg 独立连接或 get_engine()）
    - select KnowledgeDocument where deleted_at is None and is_active == True and content is not None
    - chunk_by_headers(doc.content, source_id=str(doc.id)) → index_chunks(chunks, doc_id=str(doc.id))
    - KnowledgeDocument 模型位于 backend/app/modules/knowledge/models.py，需先 Read 确认字段名
