# Sisyphus-Y 测试用例平台

## What This Is

Sisyphus-Y 是面向数据中台场景的 AI 驱动企业级功能测试用例自动生成平台。核心流程：需求录入 → AI 需求分析 → 测试点确认 → 用例生成 → 用例库管理。通过显式拆分「测什么」（M04 测试点确认）和「怎么测」（M05 用例步骤），在评审节点前置拦截方向错误。

## Core Value

让测试工程师从手写用例的重复劳动中解放，专注于测试策略和质量判断——平台生成草稿，人负责决策。

## Context

**技术基础（已验证可用）：**
- FastAPI + SQLAlchemy 2.0 async，模块化 DDD 结构，23 个业务模块
- 统一 AI 适配层（`app/ai/`），支持 7 种 LLM 提供商
- 前端 Next.js 16 App Router + Zustand + shadcn/ui
- 向量库 Qdrant，文档解析 UDA 引擎

**当前主要技术风险（来自 codebase 映射）：**
- Celery worker 任务均为 stub，长任务无法后台执行
- SSE 解析错误被 silent swallow，流中断无通知
- DB 连接池未限制，高并发 SSE 下会耗尽
- RAG 向量维度校验缺失，换模型后易出现零召回

**历史数据：**
- `[清洗后数据]/` 目录下有真实业务历史用例，需经审查才能入向量库

## Constraints

- **技术栈**：前端 bun + Next.js 16，后端 uv + Python 3.12，禁止切换
- **UI Token**：禁止硬编码色值，统一用 `sy-*` Tailwind class
- **数据安全**：核心业务数据全部软删除，禁止物理删除
- **AI 层**：所有 Prompt 只能写在 `engine/` 或 `ai/prompts.py`，service/router 层禁止
- **验收标准**：核心主链路端到端零报错；AI 审查通过率 ≥ 85%；页面刷新数据零丢失

| 决策 | 理由 | 结果 |
